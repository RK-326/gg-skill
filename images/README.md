# images/ — стиль картинок и генерация

Отвечает на вопрос «в каком стиле картинки и как их делать», чтобы новые
иллюстрации совпадали со старыми, а не выглядели каждый раз по-новому.

## Главные правила процесса
1. Картинки — ОТДЕЛЬНЫЙ этап (этап 5), только после готового текста.
2. НИКОГДА не генерировать по одной через UI. Генерация батчем через API
   (`../scripts/gen_images.py`), сразу много.
3. На каждую метку `[КАРТИНКА: ОПИСАНИЕ]` — 2-3-5 вариантов (примерно одна картинка
   на N страниц).
4. Перед генерацией Claude смотрит в эту папку: на `reference/` (как выглядели
   принятые картинки) и на `prompts/` (какие промты сработали раньше). Каждый новый
   запрос строится на лучшем предыдущем результате — это «компаундинг».
5. Рената отсматривает варианты локально, плохие удаляет, хорошие оставляет.
   Claude вставляет принятые в документ (размеры/рамки/тень — по `../template/design-spec.md`).
6. Принятые варианты и их промты сохраняются обратно в `reference/` и `prompts/` —
   так база с каждым документом обогащается.

## Стиль (фирменный вид GG) — выбран Ренатой
- **Плоская векторная иллюстрация (flat design) со сценкой и персонажами в действии**
  (студент, профессор и т.п.). Это утверждённый стиль для всех картинок документов.
- Палитра в тон документу: тёмно-синий 13445D, акцент 1A5CA8, светлый/белый фон.
- Чисто, современно. Без кислотных цветов, без стоковой пошлости, без 3D/фото.
- **БЕЗ ТЕКСТА на картинке** — ни букв, ни слов, ни подписей, ни цифр (модель их
  коверкает; пример: проскакивало «INTERNOSHIP»). Весь текст идёт в обычный текст документа.
- Финально в документе: скругление + рамка D5DBDB + общая мягкая тень.
- Размещение: ОДНА картинка после заголовка КАЖДОГО раздела (в магистратуре/бакалавриате
  17/18 разделов → столько же картинок).

## Где что лежит
- `reference/` — принятые картинки-образцы (пополняется по мере работы).
- `prompts/` — сохранённые удачные промты (`prompts/starter.md` для старта).
- Диаграммы/таблицы-иллюстрации, которые лучше считать кодом, делает
  `../scripts/gen_illustrations.py` (matplotlib), а не генеративная модель.

## Как сгенерировать (батч)
```bash
cd "$HOME/Documents/Global Generation/gg-skill"
source .env
python3 scripts/gen_images.py --prompt "схема: путь абитуриента от теста до подачи, стиль GG, палитра 13445D/1A5CA8" --n 3 --out images/_drafts
```
Черновики падают в `images/_drafts/` (gitignored). Отсмотрели → лучшее переносим в
`reference/`, промт дописываем в `prompts/`.
