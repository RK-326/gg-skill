# template/ — каркас и эталон документа

Эта папка отвечает на вопрос «как должен выглядеть готовый документ и из чего
он собирается». Здесь три вещи: каркас (порядок блоков), точная дизайн-спека и
чек-лист перед сдачей.

## Что должно лежать в этой папке
- `design-spec.md` — точные значения вёрстки (шрифты, цвета, плашки, тени, сноски). ЕСТЬ.
- `checklist.md` — что проверить перед сдачей. ЕСТЬ.
- `example.docx` — ГОТОВЫЙ документ-образец. Положить сюда копию эталона:
  `~/Documents/Global Generation/обновленные pdf/Поступление на бакалавриат США _ОБНОВЛЕНО_v4.docx`
  (см. команду ниже).
- `blank.docx` — ПУСТОЙ каркас-болванка с теми же стилями/колонтитулами/водяным
  знаком, но без контента. Из неё начинается каждый новый документ.
  (Делается один раз: открыть example.docx, удалить контент, сохранить как blank.docx.)

## Каркас документа (строгий порядок блоков)
1. Title (Montserrat Black 30)
2. Подзаголовок (italic)
3. Hero-иллюстрация (rounded + тень, по центру)
4. «Кратко о документе» — заметка italic sz18 13445D
   («Данные проверены в <месяц год>… ключевые цифры сверяйте на сайтах»)
5. Светлая карточка «Как устроен этот гайд» — 4-6 крупных тем; в строке
   «Раздел N» (bold + underline, цвет 1A5CA8) в начале, затем тема (bold 13445D),
   описание, и СПРАВА номер страницы через таб с точечным заполнителем
   (`<w:tab w:val="right" w:leader="dot" w:pos="10350"/>`)
6. (pageBreak) «Об авторе» — заголовок 13445D + фото (rounded + тень) слева + текст справа
7. Разделы — каждый Heading1 (pageBreakBefore) → плашка ВАЖНО (синяя) в начале →
   текст/картинки → плашка «Коротко по разделу» (зелёная) в конце

Точные значения каждого блока — в `design-spec.md`.

## Положить эталон в папку (один раз)
```bash
cp "$HOME/Documents/Global Generation/обновленные pdf/Поступление на бакалавриат США _ОБНОВЛЕНО_v4.docx" \
   "$HOME/Documents/Global Generation/gg-skill/template/example.docx"
```
.docx в git не коммитятся (см. .gitignore), но в папке лежать должны — это
живой образец, с которым Claude сверяется на этапе сборки.
