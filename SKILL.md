# SKILL: gg-skill — сборка документов Global Generation

Это НЕ один большой файл с правилами. Это оркестратор: он ведёт Claude по этапам
и на каждом этапе отправляет к нужной папке. Сами правила лежат отдельно
(`rules/`, `template/`, `tables/`, `images/`, `factcheck/`) — так их легче править
и так документ не «разъезжается».

Запуск: «Продолжаем gg-skill».

## Принципы (читать всегда)
1. Один документ за раз, end-to-end, до конца — потом следующий. НЕ генерировать
   десять штук параллельно (получается мусор).
2. Claude НЕ переписывает один .md файл бесконечно. Рената — оператор: ведёт Claude
   от этапа к этапу. Правки идут в нужную папку правил, а не в общий ком.
3. Рамка задачи для старых документов: усиливать, а не переписывать. Подробно —
   `rules/general-rules.md`.

## Этапы (по одному документу)

### Этап 0 — какой документ
Открыть `content/content-plan.md`, найти первый со статусом «не готов», подтвердить
у Ренаты. Прочитать исходный PDF из `~/Documents/Global Generation/ПДФ на портале/`.

### Этап 1 — общая картина и структура (ГЕЙТ)
Прочитать старый документ-образец (tone of voice). Предложить заголовок,
10-13 разделов, число картинок, источники. ДОЖДАТЬСЯ аппрува до ресёрча.
Правила текста и стиля: `rules/general-rules.md`.

### Этап 2 — фактчек
Прогнать утверждения через Perplexity по критериям `factcheck/factcheck-criteria.md`.
Скрипт: `scripts/factcheck_perplexity.py`. 3+ источника на факт, приоритет
первоисточникам, все ссылки проверены. Цикл, пока факты не станут чистыми.

### Этап 3 — текст
Усилить (не переписать) по `rules/general-rules.md`. Сноски с источниками в тексте.
Метки `[КАРТИНКА: ОПИСАНИЕ]` оставлять, картинки пока НЕ вставлять.

### Этап 4 — сборка .docx
Дизайн строго по `template/design-spec.md`. Таблицы — `tables/tables-standard.md`.
Шаблон-каркас — `template/README.md`. Сборка: `scripts/build_docx.py`.
Перед сдачей пройти `template/checklist.md` и проверить ВИЗУАЛЬНО (рендер в PNG).

### Этап 5 — картинки (отдельный этап)
Только после готового текста. Стиль и процесс — `images/README.md`.
Генерация батчем через `scripts/gen_images.py` (OpenRouter, Gemini image),
2-3-5 вариантов на метку. Рената отсматривает локально, плохие удаляет,
хорошие Claude вставляет и сохраняет обратно в `images/reference/` (компаундинг).

### Этап 6 — финал
Финальная визуальная проверка, экспорт PDF, отдать Ренате и Льву.

## Что где лежит
- `rules/general-rules.md` — общие правила для ВСЕХ документов (стиль, тире, табличный перенос, висячие слова, нумерация).
- `rules/essay-documents.md` — для документов с эссе: обязательный разбор «почему эссе сильное» + таблица приёмов.
- `template/` — эталонный документ, пустой каркас, дизайн-спека, чек-лист.
- `tables/tables-standard.md` — стандарт оформления таблиц.
- `images/` — стиль картинок, референсы, сохранённые промты, генерация.
- `factcheck/` — критерии качества для Perplexity.
- `scripts/` — рабочие скрипты (фактчек, иллюстрации, сборка, генерация картинок).
- `content/content-plan.md` — список из 21 документа и статусы.
- `output/` — здесь собираются .docx (в git не коммитится).
